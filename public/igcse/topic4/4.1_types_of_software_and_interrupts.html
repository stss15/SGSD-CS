The Big Picture

Hardware

The physical components (Engine).

Software

The programs that control the hardware (Driver).

Key Idea: Hardware is useless without Software to tell it what to do.

Two Families of Software

System Software

Programs that manage the computer's hardware and provide a platform for applications.

  • Operating System (OS)
  • Utilities (Antivirus)
  • Drivers
  • Translators (Compilers)

Application Software

Programs that allow the user to perform specific tasks.

  • Word Processor
  • Spreadsheet
  • Games
  • Web Browser
Deep Dive: Application Examples

Application Software Examples

Word Processor
Manipulate text, format documents (Word, Docs).
Spreadsheet
Organize numbers, formulas, graphs (Excel, Sheets).
Database
Store, query, and analyze data (Access, SQL).
Control
Interface with sensors to control devices.
Photo Editing
Manipulate digital images (Photoshop).
Video Editing
Cut, arrange, and enhance video clips.

The Operating System (OS)

Operating System

Software running in the background that manages basic functions and provides a platform for applications.

Examples: Windows, macOS, Linux, Android, iOS.

Deep Dive: OS Functions

Key Functions of the OS

1. HCI
Human Computer Interface (GUI/CLI).
2. Memory Mgmt
Allocates RAM to programs.
3. Multitasking
Shares CPU time between processes.
4. Security
Logins, passwords, permissions.
5. File Mgmt
Save, delete, move files.
6. Hardware
Manages peripherals via Drivers.

Deep Dive: HCI

CLI (Command Line)

  • Type commands.
  • Fast, low memory.
  • Hard to learn.
  • Used by technicians.

GUI (Graphical)

  • WIMP (Windows, Icons, Menus, Pointers).
  • Easy to use (intuitive).
  • Uses more memory.
  • Used by everyday users.

Utility Software

Programs that help manage, maintain, and control computer resources.

Virus Checkers
Scan for malware, quarantine infected files.
Defragmentation
Reorganize files on HDD to speed up access.
Backup
Create copies of data (Local or Cloud).
Security
Firewalls, Encryption.
Screensavers
Prevent screen burn (old) or lock screen (security).

Interrupts

Interrupt

A signal sent to the CPU to stop the current process and handle a high-priority task.

Why? To allow multitasking and handle hardware events (e.g., printer out of paper).
Deep Dive: The Process

The Interrupt Process

1. Normal Operation
CPU is executing a program.
2. Interrupt Signal
A device (e.g., Printer) sends a signal.
3. Pause
CPU pauses current task. Saves state (registers) to stack.
4. Service
CPU runs the Interrupt Service Routine (ISR).
5. Resume
CPU restores saved state and continues original task.

Buffers

How do we handle speed differences?

The Problem

CPU is super fast. Printer is slow.

The Solution: Buffer

A temporary memory area. CPU fills the buffer, then goes back to work. Printer empties the buffer slowly.

Link: When buffer is empty -> Interrupt sent to CPU to refill it.

  • System: OS, Utilities, Drivers.

Question 1

"Application software can run without any system software."

Answer: False

Applications rely on the OS, drivers, and utilities to access hardware resources.

Question 2

"Interrupts stop the current task permanently."

Answer: False

The CPU services the interrupt, restores the saved state, and resumes the original task.

Question 3

"Defragmentation is a utility that can speed up disk access."

Answer: True

It reorganises files so they are stored contiguously, reducing seek time.
