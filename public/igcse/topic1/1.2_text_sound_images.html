What Is a Character Set?

Character Set

A list of characters matched to numeric code points so text can be stored in binary (e.g., ASCII, Unicode).

Why It Exists
  • Standard ASCII: 7 bits, 128 codes (English letters, digits, punctuation, control codes).
  • Extended ASCII: 8 bits, 256 codes (adds symbols/accents but not consistent worldwide).
  • Unicode: global code points (e.g., U+0041 = A, U+1F600 = emoji) shared across platforms.
Clarify

A character set lists code points; an encoding (ASCII, UTF-8/16/32) decides how many bytes are used.

Why shift to Unicode? To cover every language and emoji consistently.

UTF-16 often 2 bytes; UTF-32 always 4 bytes.

How Do We Store Sound?

Digital Audio

Sound captured as regular samples (sample rate) with each sample stored using a fixed number of bits (sample resolution).

Why It Exists
  • Converts continuous waves into data computers can process
  • Enables editing, streaming, and compression
Question

What happens to quality and size if we double the sample rate?

Quality ↑ (captures higher frequencies); file size ↑ proportionally.

Sampling Parameters

Sample rate: samples per second (Hz). Must be at least 2x the highest frequency (Nyquist).

Resolution/bit depth: bits per sample; higher reduces quantisation error.

Channels: mono = 1, stereo = 2 (or more).

Clarify

Doubling sample rate or bit depth doubles file size if other factors stay constant.

Changing sample rate does not change pitch; it captures or misses frequencies.

How Does Sampling Work?

File Size (bits): sample rate x resolution x channels x duration

Bytes = (rate x res x ch x seconds) / 8

Sampling Example

Example: 44.1 kHz, 16-bit, stereo, 10 s

44,100 x 16 x 2 x 10 = 14,112,000 bits (~1.76 MB)

Try now: 22.05 kHz, 8-bit, mono, 5 s

22,050 x 8 x 1 x 5 = 882,000 bits (~110.25 KB)

How Do We Store Images?

Bitmap Image

A grid of pixels, each pixel encoded with a colour value using a fixed number of bits (colour depth).

Why It Exists
  • Pixels map directly to screen dots
  • Simple math for size and scaling
Question

What changes if we double the colour depth?

More colours, smoother gradients, but file size doubles.

Bitmap vs Vector

Bitmap: stores every pixel's colour; can pixelate when scaled up; great for photos.

Vector: stores shapes/formulas; scales cleanly; great for logos/icons.

Clarify

Vector files can be tiny for simple art; bitmaps grow with resolution and colour depth.

Question 1

"UTF-8 always uses 1 byte per character."

Answer: False

UTF-8 is variable length: ASCII stays 1 byte but other scripts/emojis need 2-4 bytes.

Question 2

"Doubling the sample rate doubles the sound file size if everything else is constant."

Answer: True

File size is directly proportional to sample rate (and resolution, channels, duration).

Question 3

"Increasing colour depth reduces the image file size."

Answer: False

Higher colour depth uses more bits per pixel, increasing size.

Question 4

"Vector images pixelate when scaled up."

Answer: False

Vectors redraw shapes mathematically, so they stay sharp; bitmaps pixelate when enlarged.
