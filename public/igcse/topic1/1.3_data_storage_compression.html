What Are Bits, Bytes, and Prefixes?

Bit / Byte

Bit is a single 0 or 1; byte is 8 bits; nibble is 4 bits.

Binary Prefixes (IEC)
  • 1 KiB = 2^10 = 1,024 bytes
  • 1 MiB = 2^20 = 1,048,576 bytes
  • 1 GiB = 2^30 = 1,073,741,824 bytes
  • Use for RAM/ROM and syllabus questions.
Clarify

KB/MB/GB (decimal) use powers of 10; KiB/MiB/GiB use powers of 2. Do not mix them.

Answer: 68,719,476,736 bytes

How Big Is a Bitmap Image?

Formula (bits): width x height x colour depth

Bytes = bits / 8

Why It Matters

Lets you budget storage and compare formats before taking photos or downloading assets.

Example: 1024 x 1080 at 32-bit colour

Press Down to see each step.

Step 1: Pixels = 1024 x 1080 = 1,105,920

Step 2: Bits = 1,105,920 x 32 = 35,389,440

Step 3: Bytes = 35,389,440 / 8 = 4,423,680

Result: ~4.22 MiB (before compression)

Clarify: increasing resolution or depth multiplies size.

Try It: Image Size

Question: 400 x 300 at 8-bit colour. What is the raw size?

Answer: 400 x 300 x 8 = 960,000 bits = 120,000 bytes (~117.2 KiB)

How Big Is a Sound File?

Formula (bits): sample rate x resolution x channels x duration

Divide by 8 for bytes; by 1,048,576 for MiB.

Clarify

Doubling sample rate or bit depth doubles size (if channels and duration stay the same).

Channels multiply size (stereo = x2).

Example: 44.1 kHz, 16-bit, stereo, 60 minutes

Press Down to see each step.

Step 1: Bits (mono) = 44,100 x 16 x (60 x 60) = 2,540,160,000

Step 2: Stereo bits = 2,540,160,000 x 2 = 5,080,320,000

Step 3: Bytes = 5,080,320,000 / 8 = 635,040,000

Result: ~605 MiB

Uncompressed audio is large; compression reduces it.

Try It: Sound Size

Question: 22.05 kHz, 8-bit, mono, 30 s

Answer: 22,050 x 8 x 1 x 30 = 5,292,000 bits = 661,500 bytes (~646 KB)

What Is File Compression?

Compression

Reducing file size to save storage and transfer time.

Why Compress?
  • Store more files on limited media.
  • Speed up downloads/uploads and streaming.
  • Lower costs for backups and hosting.
Compression Ratio

original size / compressed size (higher is better).

Lossless vs Lossy

Lossless

  • Keeps every bit; perfect reconstruction.
  • Examples: RLE, Huffman, ZIP, PNG, FLAC.
  • Use for text, code, medical images, and archival audio.

Lossy

  • Discards less noticeable data for higher compression.
  • Examples: JPEG, MP3, AAC, MP4.
  • Use for streaming media; avoid for source files you must edit.

How does run-length encoding (RLE) shrink data?

Step 1: Find runs of repeated values (e.g., 000000).

Step 2: Store value + count instead (0 x 6).

Result: Fewer bits if data has long repeats (simple graphics, icons).

Clarify: RLE is lossless; no detail is removed.

True or False?

Answer these misconception checks in your book:

  1. "KB and KiB mean the same thing."
  2. "Doubling sample rate halves the audio file size."
  3. "Lossless compression keeps every bit of the original."
  4. "JPEG is a good format for compressing source code."

Press Down to reveal answers

Question 1

"KB and KiB mean the same thing."

Answer: False

KB is decimal (1,000 bytes); KiB is binary (1,024 bytes).

Question 2

"Doubling sample rate halves the audio file size."

Answer: False

Size is proportional to rate; doubling rate doubles size (if other factors stay the same).

Question 3

"Lossless compression keeps every bit of the original."

Answer: True

Lossless can reconstruct the exact original data.

Question 4

"JPEG is a good format for compressing source code."

Answer: False

JPEG is lossy and for images; text/code needs lossless formats (ZIP, plain text).
